-Milestone: è un obbiettivo, tutte le attività (Issue) necessarie per raggiungere quell'obiettivo vengono raccolte dentro la stessa milestone.
se hai centinaia di issue, quali fai a sapere quali servono per arrivare alla prima versione funzionante? con una Milestone.
quando tutte le issue della target milestone sono chiuse, Forgejo ti mostra per quel milestone '100% Completed' ora puoi rilasciare la version e.g. v1.0.0
quindi le milestone è il gruppo di lavoro (che contiene N Issues) che serve ad arrivare a quella versione.

🔥-Semantic versioning(SemVer):
MAJOR: cambiamenti che rompono la compatibilità (v1 → v2)
MINOR: nuove funzionalità retrocompatibili (v1.2 → v1.3)
PATCH: solo bugfix (v1.2.0 → v1.2.1)
convenzione utilizzata in tutto il mondo per indicare quanto è cambiato un software e se l'aggiornamento può rompere la compatibilità.
la forma è 'vMAJOR.MINOR.PATCH' e.g. v1.4.7  , patch=fissato il bug, minor=si aggiunge una nuova funzionalita (non hai rotto niente, e patch torna a 0), major=hai rotto la compatibilita(le vecchie configurazioni non funzionano piu, questo è un breaking change )  
!!ogni volta che raggiungi una versione importante, crei un 🔥Tag e una Release(sono le releases notes):
git tag v1.0.0
e la 🔥Release che descrive le novita
Release v1.2.0
✓ Hybrid Search
