рҹ”Ҙ-Semantic versioning(SemVer):
MAJOR: cambiamenti che rompono la compatibilitГ  (v1 вҶ’ v2)
MINOR: nuove funzionalitГ  retrocompatibili (v1.2 вҶ’ v1.3)
PATCH: solo bugfix (v1.2.0 вҶ’ v1.2.1)
convenzione utilizzata in tutto il mondo per indicare quanto ГЁ cambiato un software e se l'aggiornamento puГІ rompere la compatibilitГ .
la forma ГЁ 'vMAJOR.MINOR.PATCH' e.g. v1.4.7  , patch=fissato il bug, minor=si aggiunge una nuova funzionalita (non hai rotto niente, e patch torna a 0), major=hai rotto la compatibilita(le vecchie configurazioni non funzionano piu, questo ГЁ un breaking change )

## Il flusso completo di una feature
```bash
# 1. Parti sempre da develop aggiornato
git checkout develop
git pull origin develop

# 2. Crea la feature branch
git checkout -b feature/rag-web-search

# 3. Lavori, fai commit piccoli e frequenti
git add app/rag/agents/web_agent.py
git commit -m "feat: aggiunge web agent con Tavily"
git add tests/unit/test_web_agent.py
git commit -m "test: aggiunge test per web agent"

# 4. develop ГЁ andato avanti? Aggiorna con rebase
git fetch origin
git rebase origin/develop

# 5. Push della feature
git push origin feature/rag-web-search

# 6. Su Forgejo: apri Pull Request feature/rag-web-search вҶ’ develop
#    Il team fa code review, la CI gira i test

# 7. Merge approvato вҶ’ "Create merge commit"
#    develop ora contiene la feature

# 8. develop viene deployato automaticamente in DEV dal runner

# 9. Cancella la feature branch (locale e remota)

git push origin --delete feature/rag-web-search
```

I tre ambienti non sono solo cartelle diverse. Sono isolati a ogni livello:

| Livello | Dev | Staging | Production |
|---|---|---|---|

| Rete Docker | rag_dev_net | rag_staging_net | rag_prod_net |
| Porte host | 8000-8099 | 8100-8199 | 8200-8299 |
| Volumi dati | /srv/data/dev | /srv/data/staging | /srv/data/prod |
| Branch Git | develop | staging | main (tag) |
| Sottodominio | dev.dominio.it | staging.dominio.it | app.dominio.it |
| Dati | finti/campione | copia anonimizzata | reali clienti |
| Deploy | automatico | manuale su tag | manuale approvato |
